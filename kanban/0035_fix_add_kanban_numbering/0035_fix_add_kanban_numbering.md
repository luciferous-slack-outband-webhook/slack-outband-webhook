# add-kanban 採番バグの修正

## 目的
`/add-kanban` スキルの採番ロジックが壊れており、既存タスクがあるにもかかわらず常に `0001` を返してしまう。採番ロジックを修正してスキルを正常に動作させたい。

## 要望
採番手順（手順2）で Glob を使っているが、スキル起動時のカレントディレクトリがスキルディレクトリ（`.claude/skills/add-kanban/`）になっているため `kanban/` が見つからず空を返す。

Glob の代わりに Bash で `ls kanban/` を実行する方式に変更し、採番が正しく行われるよう修正してください。

- 完了日時: 2026-04-21T21:52:06+09:00
- `.claude/skills/add-kanban/SKILL.md` 手順2の採番ロジックを Glob から Bash（`ls kanban/ 2>/dev/null | grep -E '^[0-9]{4}_' | sort | tail -1`）に変更した
- 動作確認: コマンドが `0035_fix_add_kanban_numbering` を正しく返すことを確認。次タスクは `0036` に採番される

## プラン

`.claude/skills/add-kanban/SKILL.md` 手順2のみを以下のように差し替える。

**変更後:**
