## タスク概要

採番手順（手順2）で Glob を使っているが、スキル起動時のカレントディレクトリがスキルディレクトリ（`.claude/skills/add-kanban/`）になっているため `kanban/` が見つからず空を返す。Glob の代わりに Bash で `ls kanban/` を実行する方式に変更し、採番が正しく行われるよう修正する。

---

## 調査結果

### バグの再現経緯

`/add-kanban` スキルを実行した際、Glob ツールで `pattern: kanban/[0-9][0-9][0-9][0-9]_*/`（path 省略）を呼び出したところ「No files found」が返った。実際には `kanban/` 配下に `0000_cannot_git_pull` 〜 `0034_complete_new_kanban_structure` の35ディレクトリが存在していた。

ユーザーが `ls kanban/` を実行したところ正しく35ディレクトリが表示されたため、Bash のカレントディレクトリはリポジトリルート（`/Users/yuta/space/projects/luciferous-slack-outband-webhook/00_repos/slack-outband-webhook`）で動作していることを確認。

### スキルのベースディレクトリ確認

```
/Users/yuta/.../slack-outband-webhook/.claude/skills/add-kanban/
└── SKILL.md  （kanban/ サブディレクトリは存在しない）
```

スキルには `SKILL.md` のみが存在し、`kanban/` サブディレクトリは存在しない。スキル起動時に Glob の探索起点がスキルディレクトリになった場合、`kanban/` は存在しないため空マッチになると考えられる。

### SKILL.md 手順2（変更前）

```
2. **採番**: Glob で `kanban/[0-9][0-9][0-9][0-9]_*/` を取得する。ディレクトリ名から番号部分を読み取り、最大番号 +1 を4桁0パディングで次の番号とする（例: 最大が 0029 なら次は `0030`）。
```

### Bash vs Glob のトークン消費量

ユーザーから「トークン消費量が少ない方を使いたい」という要望があり検討した。

- **Glob 全件取得**: 35件 × フルパス（約100文字）≒ 3500トークン
- **Bash `ls | grep | sort | tail -1`**: 出力1行のみ（約35文字）≒ 35トークン

→ Bash の方が約100倍少ない。タスク数が増えるほど差が開く。Bash 採用を決定。

## 実装プラン

`.claude/skills/add-kanban/SKILL.md` 手順2のみ変更。他は一切触らない。

**変更後の手順2:**

```
2. **採番**: Bash で `ls kanban/ 2>/dev/null | grep -E '^[0-9]{4}_' | sort | tail -1` を実行し、リポジトリルートの `kanban/` 配下にあるタスクディレクトリから最大番号を取得する。出力されたディレクトリ名の先頭4桁を最大番号として読み取り、+1 を4桁0パディングで次の番号とする（例: 出力が `0029_foo` なら次は `0030`）。出力が空の場合は `0001` を使用する。
```
